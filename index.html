New Buisness To Be Discussed

  1. Grading criteria for the project:
    • Features, our product should have all needed features developed to be easily used by the customer
    • Process, the way in which our product is developed should be in a organized maner
    • Quality, our product should be checked for quality, with bugs eliminated and features optimized
  2. Testing:
    • Each feature that is developed should be tested by its developer
    • We also believe that we should assign someone to be in charge of testing new features to find bugs
  3. Documentation:
    • Our code base should be well commented, with discriptions for each function
    • JS docs is a recommended way to do this
  4. Progression:
    • Our progression chart on github should be as linear as possible
    • A large spike at the end may happen, but to minimize stress avoid this
  5. Stand-up meetings:
    • Must reserve a time slot
    • At least one team leader must be present for these
    • Discussion will focus on project progress that week
  6. Role assignment:
    • Currently we do not have a designer
    • At least one member should be assigned to this missing role
